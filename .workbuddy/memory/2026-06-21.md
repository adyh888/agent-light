## 修复：灯光模式关闭时圆圈应显示灰色

**文件**: `web/src/components/StatusPanel.vue`

**问题**: 在 StateConfigPanel 中将某状态的灯光模式设为 `off`（关闭）后，StatusPanel 顶部的灯光圆圈仍然显示原来的绿/黄/红颜色，没有变成灰色。

```js
  backgroundColor: currentMode.value === 'off' ? '#d1d5db' : (LIGHT_COLORS[currentLight.value] || '#4ade80'),
}))
```

## 功能：WorkBuddy 接入 Agent Light 红绿灯联动

**需求**: 让 WorkBuddy 在工作时自动控制 USB 三色灯，反映 AI 工作状态。

**实现**: 创建 WorkBuddy Skill（`~/.workbuddy/skills/skill_1782034850275053/SKILL.md`），包含：
- 6 种状态映射：thinking(思考中) / busy(忙碌) / completed(完成) / waiting_for_user(等待) / idle(空闲) / off(关灯)
- 每种状态对应的 curl 命令，静默执行（`> /dev/null 2>&1 &`）
- 触发时机说明：开始工作→thinking，耗时操作→busy，任务完成→completed，提问→waiting_for_user
- 服务地址 `http://localhost:3777`，Web 面板同地址

**服务**: agent-light 已启动（PID 16810，端口 3777），串口已连接 `/dev/tty.wchusbserial2120`

**测试**: thinking/completed/idle 三种状态 API 调用均成功，灯光响应正常。

## 修复：Agent Light Skill 灯不响应

**问题**: Skill 中 curl 命令使用了 `> /dev/null 2>&1 &` 后台重定向，在沙箱环境中不稳定，导致灯不亮。

**修复**: 更新 Skill（v1.1.0）：
- 移除所有 `> /dev/null 2>&1 &` 后台重定向
- 改为通过 Bash 工具直接前台执行 curl 命令
- 每次只发一条 curl，不与其他命令拼接
- 测试确认 thinking → completed 状态切换正常，灯光响应正确

## 修复：新窗口 Skill 不自动触发

**问题**: 新开的对话窗口里，agent-light skill 不会被自动加载，导致灯没有任何反应。

**根因**: Skill 的 description 写得不够强，新窗口的 AI 看到技能列表后不知道应该在每个任务开始时主动加载这个 skill。

**修复**: 更新 Skill（v1.2.0）：
- description 改为 "MUST LOAD at the start of EVERY conversation and task"
- 正文开头加 ⚠️ 强制指令块，明确要求每次对话必须加载和执行
- 明确三步必须操作：thinking → completed → waiting_for_user
- description_zh 也改为"每次对话开始时必须加载"

**关键教训**: Skill 的 description 是新窗口 AI 判断是否加载的唯一线索，必须用强烈的措辞（MUST、每次、必须）让 AI 知道这是强制触发的。

## 修复：新窗口灯控仍不触发 — 改用全局记忆

**问题**: Skill v1.2.0 升级后，新窗口仍然不触发灯控。根因是 Skill 靠关键词触发，用户任务描述里没提到"灯"就不会加载 skill。

**真正解决方案**: 创建用户级全局记忆文件 `~/.workbuddy/MEMORY.md`，这个文件在每次新对话开始时自动注入 AI 上下文，不依赖关键词触发。

**内容**: 写入强制灯控规则（thinking/completed/waiting_for_user 三步），标注为"每个任务的强制操作"。

**关键教训**:
- Skill 是关键词触发，不可靠用于"每次都必须执行"的操作
- `~/.workbuddy/MEMORY.md` 是全局注入的，适合放跨项目的强制规则
- 两层保险：Skill 处理灯控细节，MEMORY.md 确保每次对话都触发
